Loan Approval Prediction

Predicting Loan Status with Machine Learning

This project focused on leveraging machine learning to accurately predict loan approval status, a crucial task for financial institutions to manage risk and optimize operations. The analysis utilized two datasets: a training set ("train_data.csv") with known loan statuses and a test set ("test_data.csv") for which the loan status was to be predicted.

Project Overview

The project began with loading the datasets using the Pandas library in Python. Initial exploration involved examining the data structure and checking for duplicate entries. No duplicate rows were found in either dataset. A thorough assessment of missing values was conducted to understand data completeness. Categorical features were examined to determine the number of unique values and their distribution.

To prepare the data for machine learning, several preprocessing steps were taken. Missing values were removed by dropping rows containing them. Categorical variables were encoded using a combination of label encoding for binary variables and one-hot encoding for multi-category variables. Finally, numerical features were scaled using the StandardScaler to ensure all features contribute equally to the models.

Feature Analysis and Visualization

Visualizations were employed to gain insights into the data. Count plots illustrated the relationship between categorical variables and loan status, providing a clear view of how different categories influence approval rates. Box plots were used to identify outliers in numerical features, which could potentially skew model performance. A correlation matrix heatmap revealed the relationships between numerical variables, helping to understand potential multicollinearity.

Model Selection and Evaluation

A variety of machine learning models were evaluated for their ability to predict loan status. These included:

  • Logistic Regression
  • Decision Tree
  • Random Forest
  • Extra Trees
  • Support Vector Machine (SVM)
  • K-Nearest Neighbors (KNN)
  • Gaussian Naive Bayes
  • AdaBoost
  • Gradient Boosting Classifier
  • Dense Neural Network (MLPClassifier)

Each model's hyperparameters were tuned using GridSearchCV to optimize performance. The models were trained on a portion of the training data and validated on a separate validation set. Accuracy and classification reports were used to assess the performance of each model.

Results and Discussion

The analysis revealed the following validation accuracies for each model:

Model Validation Accuracy
Logistic Regression (LR) 0.8229
Decision Tree (DT) 0.8229
Random Forest (RF) 0.8125
Extra Trees (ET) 0.8125
Support Vector Machine (SVM) 0.8229
K-Nearest Neighbors (KNN) 0.8125
Gaussian Naive Bayes (GNB) 0.2917
AdaBoost (AdB) 0.8125
Gradient Boosting Classifier (GBC) 0.8021
Dense Neural Network (DNN) 0.5625

Logistic Regression, Decision Tree, and Support Vector Machine achieved the highest validation accuracy (approximately 82%). Gaussian Naive Bayes and the Dense Neural Network showed relatively lower performance compared to the other models.

Choice of Model

Given their comparable performance and relative simplicity, Logistic Regression, Decision Tree, or SVM would be suitable choices for this loan approval prediction task. While all three performed similarly, the choice may depend on other factors:

  • Logistic Regression: Offers interpretability, as coefficients can be examined to understand feature importance.
  • Decision Tree: Provides a clear decision-making process but can be prone to overfitting if not pruned properly.
  • SVM: Effective in high-dimensional spaces but can be less interpretable than Logistic Regression or Decision Trees.

In this analysis, the models showed very similar results.

Conclusion

This analysis demonstrates the effectiveness of machine learning in predicting loan approval status. The insights gained from data exploration and feature analysis, combined with rigorous model evaluation, provide a solid foundation for developing a loan approval prediction system. Further improvements could involve exploring more advanced feature engineering techniques or gathering additional data to enhance model accuracy and robustness.
