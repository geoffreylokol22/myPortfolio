Medical Fraud Detection with Machine Learning

Introduction

This project focused on developing a machine learning model to detect potential fraud within the healthcare industry. Medical fraud is a serious issue that inflates costs and compromises the quality of care. My goal was to create a tool that could accurately identify fraudulent claims and activities.

Project Overview

The project involved a comprehensive analysis of healthcare data, including provider information, beneficiary details, and claim records (both inpatient and outpatient). I used Python and various machine learning libraries to preprocess the data, engineer relevant features, train classification models, and evaluate their performance. The key challenges included handling imbalanced data and the complexity of healthcare billing patterns.

Technologies Used

  • Python
  • pandas
  • numpy
  • scikit-learn
  • matplotlib
  • seaborn
  • xgboost
  • lightgbm

Implementation Details

Data Loading and Preprocessing

The data was loaded from Excel files into pandas DataFrames. Missing values were addressed using appropriate imputation techniques, and columns with excessive missing data were removed. Categorical variables were encoded to make them suitable for machine learning algorithms.

Feature Engineering

I engineered new features to enhance the model's ability to detect fraud. These included metrics like claim duration, length of stay, beneficiary claim frequency, and indicators of past provider fraud.

Model Selection and Training

I experimented with several classification models, including Logistic Regression, Random Forest, XGBoost, and LightGBM. The models were trained and validated using techniques like train-test split and cross-validation.

Addressing Class Imbalance

Given the imbalanced nature of the dataset (fewer fraudulent cases than non-fraudulent), I employed techniques to mitigate bias and ensure the model effectively identified fraudulent instances.

Results and Outcomes

The LightGBM model demonstrated strong performance in detecting medical fraud. On the validation set, the model achieved an overall accuracy of 84%. The ROC AUC score was 0.8038, indicating good discriminatory power. The specificity (True Negative Rate) was 0.94, and the sensitivity (Recall / True Positive Rate) was 0.67.

These results suggest that machine learning can be a valuable tool for identifying potential fraud, leading to cost savings and improved efficiency in healthcare administration.
